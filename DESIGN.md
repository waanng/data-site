# DATA LABS 设计说明

## 风格定位

DATA LABS 采用明亮、克制、专业的数据作品集风格。整体目标不是做成炫技型大屏，而是让项目内容、指标和 Notebook 入口清楚易读，适合数据分析师展示作品与沉淀案例。

## 页面结构

```text
首页
├── 顶部导航
├── Hero: 项目定位 + 主入口
├── 关键指标
├── 三个项目卡片
├── 技术栈
└── 页脚

项目列表页
├── 项目集合说明
└── 三个项目入口卡片

项目详情页
├── 项目概览
├── 核心指标
├── 分析亮点
├── 数据字段 / 流程
└── Notebook 入口

Notebook 页
├── 左侧信息栏
└── iframe Notebook 预览
```

## 设计原则

- 内容优先：项目名称、关键指标、分析价值要第一眼可见
- 明亮专业：白底、浅灰区块、蓝绿数据感，不使用重暗色背景
- 组件统一：卡片、按钮、标签、指标使用同一套圆角、阴影和边框
- 响应式兜底：桌面多列，移动端自动单列，导航可折叠
- 外链透明：Notebook 依赖 nbviewer，文档中明确标注风险

## 后续优化方向

- 将项目数据抽成 JSON，统一生成首页和项目列表
- 使用 `jupyter nbconvert` 生成本地 Notebook HTML，减少外部依赖
- 为每个项目补充真实封面图或关键可视化截图
- 添加基础 SEO 信息和社交分享图
